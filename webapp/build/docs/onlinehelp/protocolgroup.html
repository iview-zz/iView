Application Groups
 
Cyberoam iView generates reports based on application groups. The application group is a logical grouping of applications based on their functions, for example, all FTP related applications are part of FTP application group. Cyberoam iView has grouped the most common applications under 27 pre-defined application groups.
 
Each Application has an identifier in the form of protocol and port number through which it is identified. E.g., Web-Proxy application is identified through protocol TCP and port number 8080.
 
If application is not defined in iView then instead of application name, protocol and port number will be displayed in Reports. Cyberoam iView also allows the administrator to add custom applications and application groups.
.
Use System → Configuration → Application Groups page to add and manage applications in Cyberoam iView.
 
Screen Components:
•
Application Group: Name of the application group. Click and expand the application group to view or edit member applications.
•
Description: Description of the application group.
•
Add Application Button :Click to add a new application.
•
Add Application Group Button: Click to add a new application group.
•
Reset to Default Button: Click to restore default applications and application groups.

Add Custom Application
 
There are two steps to add a custom application.
 
•
Add Application
•
Add Application Identifier
 
Add Application
 
•
Super Admin or Admin privilege required to add a new application
 
 
1.
Go to System → Configuration → Application Groups and click Add Application
2.
Specify name of the application, application name can be any combination of alphanumeric characters and special characters “_”, “@” and “.”
3.
Select application group from the drop down. If the Application Group is not selected, by default, new Application is added to the “Unassigned” group
4.
Click Done to add a new application. Once the Application is added, you need to add the application identifier to identify the application. If Application identifier is not added then the application will be considered as undefined application and reports will also reflect the same.
 
Add Application Identifier
 
•
Super Admin or Admin privilege required to add an application identifier
 
Application identifier consists of application type and port which is used to identify an application uniquely.
1.
Go to System → Configuration → Application Groups and expand the Application Group tree.
2.
Click application to add application identifier.
3.
Click Add Application Identifier
4.
Select application type as TCP or UDP.
5.
Select port type as port or port range.
6.
If port range is selected as port type then specify to and from value for port range. To port value must be greater than from port value.
7.
Click Done to create a new application identifier.
 
•
An application cannot be the member of multiple application groups. To change the group membership, first remove an application from the current group and then add in the required application group.
 
Add Application Group
 
•
Super Admin or Admin privilege required to add a new application group
 
1.
Go to System → Configuration → Application Groups.
2.
Click Add Application Group.
3.
Specify name of application group, application group name can be any combination of alphanumeric characters and special characters “_”, “@” and “.”
4.
Specify description, if required.
5.
Click the Application to be included from the ‘Unassigned Applications’ list and click to move to the 'Selected Applications' list. At least one Application is to be added. Use Ctrl+ Mouse click to select multiple applications. Selected application(s) will be the member of the newly created Application Group.
6.
Click Done to create a new application group.
Update Application
 
•
Super Admin or Admin privilege required to edit an existing application
 
1.
Go to System → Configuration → Application Groups.
2.
Expand Application Group tree and click application to be modified.
3.
Click Add Application Identifier to add a new application identifier to the application, if required.
4.
Change application group from the drop down, if required.
 
•
All fields are editable except application name.
 
Update Application Group
 
•
Super Admin or Admin privilege required to update an existing application group
 
1.
Go to System → Configuration → Application Groups and click the application group that has to be modified.
2.
Refer to Add Application Group for information on each parameter.
 
•
All fields are editable except application group name.
 
Delete Application
 
•
Super Admin or Admin privilege required to delete an application
 
1.
Go to System → Configuration → Application Groups.
2.
Expand Application Group tree.
3.
Click against application name to delete.
Delete Application Identifier
 
•
Super Admin or Admin privilege required to delete an application identifier
 
1.
Go to System → Configuration → Application Groups
2.
Expand Application Group tree.
3.
Click application to delete application identifier.
4.
Click against application identifier.
Delete Application Group
 
•
Super Admin or Admin privilege required to delete an application group
 
1.
Go to System → Configuration → Application Groups.
2.
Click against application group name to delete.
 
•
When you delete an application group, applications under that group will also be deleted.
Update Application Group membership
 
•
Super Admin or Admin privilege required to change an application group
 
There are two ways to change application group of an application:
1.
Go to System → Configuration → Application Groups and click current application group of the application.
2.
Move application to the list of unassigned application.
3.
Click Done to save the change.
4.
Refer Add Application Group and Update Application Group for details.
OR
1.
Go to System → Configuration → Application Groups and select application to be moved from current application group.
2.
Change Application Group.
3.
Click Done to save the changes.
 
Reset to Default Application Group
 
•
Super Admin or Admin privilege required to reset to default application group
 
1.
Go to System → Configuration → Application Group
2.
Click Reset to Default to restore all the application identifiers, applications and application groups to the default state. This option will delete custom created applications and application group.
