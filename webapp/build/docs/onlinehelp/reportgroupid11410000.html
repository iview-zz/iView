Reports by Mail Viruses
To view the following granular reports of a particular mail, go to Reports → Virus → Mail Viruses and drill down by clicking virus name in the graph or the virus name hyperlink in the table:
•
Top Senders (Email IDs)
•
Top Recipients (Email IDs)
•
Top Sender Hosts
•
Top Receiver Hosts
•
Top Applications
 
The granular reports page displays multiple reports in the widgets form, which can again be drilled down to view the detailed report.
 
Each widget displays report in graph as well as in tabular format.
By default, the report is displayed for the current date and all the devices. Report date or devices can be changed from the top most row of the page.
Top Senders (Email IDs) widget
Widget report displays number of connections per sender for the selected mail virus.
 
View report from Reports → Virus → Mail Viruses → Virus.
 
Bar graph displays number of connections per sender whille tabular report contains following information:
•
Sender: Email address of the sender
•
Connections: Number of connections to the sender
 
To view the following granular reports of each sender, drill down by clicking the sender email address in the graph or hyperlink in the table:
•
Top Recipients i.e. number of connections per recipient by the selected sender and mail virus
•
Top Sender Hosts i.e. number of connections per host (sender) by the selected sender and mail virus
•
Top Receiver Hosts i.e. number of connections per host (receiver) by the selected sender and mail virus
•
Top Applications i.e. number of connections per application by the selected sender and mail virus
•
Top Users i.e. number of connections per user by the selected sender and mail virus
 
The granular reports page displays multiple reports in the widgets form, which can again be drilled down to view the detailed report.

Top Applications widget

View report from Reports → Virus → Mail Viruses → Virus.

•
Application: Displays name of the application as defined in Cyberoam iView. If application is not defined in Cyberoam iView then this field will display application identifier as combination of protocol and port number. To define an undefined application or group an unassigned application please refer to Add Custom Application
