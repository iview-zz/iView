Reports for UTM Product Category : Source Host Based Usage Report : Reports by Users (Upload)
Reports by Users (Upload)
To view the following granular reports of a particular user, go to Reports → Source Host Based Usage → Top Users (Upload) and drill down by clicking user name in the graph or the user name hyperlink in the table:
•
Top Application Groups (Upload)
•
Top Destinations (Upload)
•
Top Hosts (Upload)
 
The granular reports page displays multiple reports in the widgets form, which can again be drilled down to view the detailed report.
 
Each widget displays report in graph as well as in tabular format.
By default, the report is displayed for the current date and all the devices. Report date or devices can be changed from the top most row of the page.
Top Application Groups (Upload) widget
Widget report displays amount of data uploaded and number of connections per application group by the selected user.
 
View the report from Reports → Source Host Based Usage → Top Users (Upload) → User.
Bar graph displays amount of data uploaded by each application in the group while tabular report contains following information:
•
Application: Displays name of the application as defined in Cyberoam iView. If application is not defined in Cyberoam iView then this field will display application identifier as combination of protocol and port number. To define unidentified application and group unassigned application please refer to Add Custom Application
•
Connections: Number of connections to the application group
•
Bytes: Amount of data uploaded
 
To view the following granular reports of each application in a group, drill down by clicking the application group name in the graph or hyperlink in the table:
•
Top Application (Upload) i.e. amount of data uploaded and number of connections per application by the selected user and application group
•
Top Destination (Upload) i.e. amount of data uploaded and number of connections per destination by the selected user and application group
•
Top Host (Upload) i.e. amount of data uploaded and number of connections per host by the selected user and application group
The granular reports page displays multiple reports in the widgets form, which can again be drilled down to view the detailed report.
Top Destinations (Upload) widget
Widget report displays amount of data uploaded and number of connections per destination by the selected user.
View the report from Reports → Source Host Based Usage → Top Users (Upload) → User.
Bar graph displays amount of data uploaded to each destination while tabular report contains following information:
•
Destination: Site URL or Destination IP address
•
Connections: Number of connections to the destination
•
Bytes: Amount of data uploaded
 
To view the following granular reports of each destination, drill down by clicking the destination link in the graph or hyperlink in the table:
•
Top Application (Upload) i.e. amount of data uploaded and number of connections per application by the selected user and destination
•
Top Host (Upload) i.e. amount of data uploaded and number of connections per host by the selected user and destination
 
The granular reports page displays multiple reports in the widgets form, which can again be drilled down to view the detailed report.
Top Hosts (Upload) widget
Widget report displays amount of data uploaded and number of connections per host by the selected user.
View the report from Reports → Source Host Based Usage → Top users (Upload) → User.
Bar graph displays amount of data uploaded through each host while tabular report contains following information:
•
Host: Host IP address
•
Connections: Number of connections to the host
•
Bytes: Amount of data uploaded
 
To view the following granular reports of each host, drill down by clicking the host link in the graph or hyperlink in the table:
•
Top Application (Upload) i.e. amount of data uploaded and number of connections per application by the selected user and host
•
Top Destination (Upload) i.e. amount of data uploaded and number of connections per host by the selected user and host
