Reports for UTM Product Category : Blocked Attempts : Reports by Host and Destination
Reports by Host and Destination
To view the following granular reports of a particular host and destination, go to Reports → Blocked Attempts → Top Denied Hosts→ Host → Top Destinations widget and drill down by clicking the destination in the graph or hyperlink in the table.
•
Top Applications
•
Top Users
The granular reports page displays multiple reports in the widgets which can again be drilled down to view the detailed report.
 
Each widget displays report in graph as well as in tabular format.
By default, the report is displayed for the current date and all the devices. Report date or devices can be changed from the top most row of the page.
Top Applications widget
Widget report displays number of connections per application by the selected destination and host
 
View the report from Reports → Blocked Attempts → Top Denied Hosts → Host→ Top Destinations widget→ Destination.
 
Bar graph displays number of connections to each application while tabular report contains following information:
•
Application Group: Displays name of the application as defined in Cyberoam iView. If application is not defined in Cyberoam iView then this field will display application identifier as combination of protocol and port number. To define an unidentified application and group an unassigned application please refer to Add Custom Application
•
Connections: Number of connections to the application
 
To view the list of users for the selected application, destination and host, drill down by clicking the application in the graph or hyperlink in the table.
Top Users widget
Widget report displays number of connections per user by the selected host and destination.
 
View the report from Reports → Blocked Attempts → Top Denied Hosts → Host→ Top Destinations widget→ Destination.
 
Bar graph displays number of connections to each user in the group while tabular report contains following information:
•
User: Username of the user as defined in the monitored device. If User is not defined in the monitored device then it will be considered as traffic generated by ‘Unknown’ user
•
Connections: Number of connections to the user
 
To view the list of applications for the selected user, host and destination, drill down by clicking the user name in the graph or hyperlink in the table.
