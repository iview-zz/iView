Reports by Mail Recipient
To view the following granular reports of a particular email recipient, go to Reports → Mail Usage → Top Mail Recipients and drill down by clicking recipient’s email ID in the graph or the recipient’s email ID hyperlink in the table:
•
Top Senders
•
Top Source Host
•
Top Destinations
•
Top Users
•
Top Applications
The granular reports page displays multiple reports in the widgets form, which can again be drilled down to view the detailed report.
 
Each widget displays report in graph as well as in tabular format.
By default, the report is displayed for the current date and all the devices. Report date or devices can be changed from the top most row of the page.
Top Senders widget
Widget report displays amount of data transferred and number of connections per sender for the selected recipient
 
View the report from Reports → Mail Usage → Top Mail Recipients →Recipient.
 
Bar graph displays amount of data transferred by each sender whille tabular report contains following information:
•
Sender: Email ID of the sender
•
Connections: Number of connections to the sender
•
Bytes: Amount of data transferred
 
To view sender details for the selected recipient, drill down by clicking the sender in the graph or hyperlink in the table.
Top Source Hosts widget
Widget report displays amount of data transferred and number of connections per source host for the selected recipient.
 
View the report from Reports → Mail Usage → Top Mail Recipients → Recipient.
 
Bar graph displays amount of data transferred by each source host while tabular report contains following information:
•
Source Host: IP address of the source host
•
Connections: Number of connections to the source host
•
Bytes: Amount of data transferred
 
To view source host details for the selected recipient, drill down by clicking the source host in the graph or hyperlink in the table.
Top Destinations widget
Widget report displays amount of data transferred and number of connections per destination for the selected recipient.
 
View the report from Reports → Mail Usage → Top Mail Recipients → Recipient.
 
Bar graph displays amount of data transferred to each destination while tabular report contains following information:
•
Destination: URL name or IP address of the destination
•
Connections: Number of connections to the destination
•
Bytes: Amount of data transferred
 
To view destination details for the selected recipient, drill down by clicking the destination in the graph or hyperlink in the table.
Top Users widget
Widget report displays amount of data transferred and number of connections per user for the selected recipient.
 
View the report from Reports → Mail Usage → Top Mail Recipients → Recipient.
Bar graph displays amount of data transferred by each user while tabular report contains following information:
•
User: Username of the user as defined in the monitored device. If User is not defined in the monitored device then it will be considered as traffic generated by ‘Unknown’ user
•
Connections: Number of connections to the user
•
Bytes: Amount of data transferred
 
To view user details for selected recipient, drill down by clicking the User in the graph or hyperlink in the table.
Top Applications widget
Widget report displays amount of data transferred and number of connections per application for the selected recipient.
 
View the report from Reports → Mail Usage → Top Mail Recipients → Recipient.
 
Bar graph displays amount of data transferred by each application while tabular report contains following information:
•
Application: Displays name of the application as defined in Cyberoam iView. If application is not defined in Cyberoam iView then this field will display application identifier as combination of protocol and port number. To define unidentified application or group unassigned application please refer to Add Custom Application
•
Connections: Number of connections to the application
•
Bytes: Amount of data transferred
 
To view application details for selected recipient, drill down by clicking the application in the graph or hyperlink in the table.
