Device
Cyberoam iView provides reports for multiple product categories. It helps the Super Admin to view consolidated reports and dashboards for same type of devices at one shot.
Given below is the list of product categories along with the type of devices:
•
UTM – Cyberoam, Fortinet, SonicWALL, Cisco ASA, Cisco PIX, Netfilter, Squid
•
Access Gateway – 24Online
•
EPS – eScan
•
Web Server – Apache
The administrator needs to mention the type of device while adding it to Cyberoam iView.
To generate reports based on product category, one needs to select a product category from the drop down provided on the top of the navigation pane.
Entire Cyberoam iView GUI changes according to the product category.
This section describes how to add and configure devices that communicate with Cyberoam iView.
 
Use System → Configuration → Device page to add and configure devices to communicate with Cyberoam iView.

There are two ways to add device to the Cyberoam iView:
•
Device Auto Discovery
•
Manual Device Addition
 
Device Auto Discovery
 
Cyberoam iView uses UDP protocol to discover the network device automatically. In order to send logs to Cyberoam iView, network device has to configure Cyberoam iView as a Syslog server.
 
On successful login, Super Admin will be prompted with a popup "New Device Found" if a new device is discovered; else the Main Dashboard is displayed. This prompt will be displayed every time Super Admin logs in until she takes action on the newly discovered device.
Super Admin can
•
ignore this prompt by clicking
•
accept and activate the device by providing Device Name and Device Type. Cyberoam iView will accept the logs only after device is activated.
•
accept and keep device in deactivated state. Cyberoam iView will not accept the logs if device is in deactive state.

Add Device
 
•
Super Admin privilege required to add a new device.
 
1.
Go to System → Configuration → Device and click Add.
2.
Specify device ID and device name. Device ID and device name can be any combination of alphanumeric characters and special characters “_”, “@” and “.”
3.
Specify IP address of the device.
4.
Select device type from the drop down. Possible device types are Cyberoam, FortiGate, Squid, 24Online, SonicWALL, Linux Netfilter Firewall, Cisco ASA, Cisco ASA CSC SS
5.
Specify device description, if required.
6.
Select status of the device from drop down, default status of a device is ‘Deactive’. To receive logs from the device one needs to activate the device in Cyberoam iView.
7.
Click Add to add the device to Cyberoam iView.
Activate Device
 
•
Super Admin privilege required to activate a device.
 
1.
Go to System → Configuration → Device and click ‘ ‘Active’ against device name.
2.
You can also activate the device from Update Device section. After activation, Cyberoam iView will start receiving logs from the device.
Update Device
 
•
Super Admin privilege required to update a device.
 
1.
Go to System → Configuration → Device Management and click device to be updated.
2.
Refer Add Device for information on each parameter.
 
•
All the fields except Device ID and Device Type are editable.
Deactivate Device
 
•
Super Admin privilege required to deactivate a device.
 
1.
Go to System → Configuration → Device and click ‘ ‘Deactivate’ option against device name.
2.
You can also deactivate the device from Update Device section. After deactivation, Cyberoam iView will stop receiving logs from the device.
•
To access the data of device for forensic investigations do not delete the device from Cyberoam iView, just deactivate it.
Delete Device
 
•
Super Admin privilege required to delete a device.
•
Device should not be a member of any device group.
 
1.
Go to System → Configuration → Device.
2.
Click checkbox against the device(s) to be deleted OR click Select All to delete all devices.
3.
Click Delete.
