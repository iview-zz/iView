Reports by Mail Virus and Sender Host
To view the following granular reports of a particular mail virus and sender host go to Reports → Virus → Mail Viruses → Virus → Top Sender Hosts widget and drill down by clicking the sender host in the graph or hyperlink in the table.
•
Top Senders
•
Top Recipients
•
Top Receiver Hosts
•
Top Applications
•
Top Users
 
The granular reports page displays multiple reports in the widgets form, which can again be drilled down to view the detailed report.
 
Each widget displays report in graph as well as in tabular format.
By default, the report is displayed for the current date and all the devices. Report date or devices can be changed from the top most row of the page.
Top Senders widget
Widget report displays number of connections per sender by the selected mail virus and sender host.
 
View report from Reports → Virus → Mail Viruses → Virus → Top Sender Hosts widget → Sender Host.
 
Bar graph displays number of connections per sender while tabular report contains following information:
•
Sender: Email address of the sender
•
Connections: Number of connections to the sender
 
To view the sender details for the selected sender, sender host and mail virus, drill down by clicking the sender in the graph or hyperlink in the table.
Top Recipients widget
Widget report displays number of connections per recipient by the selected mail virus and sender host.
 
View report from Reports → Virus → Mail Viruses → Virus → Top Sender Hosts widget → Sender Host.
 
Bar graph displays number of connections per recipient while tabular report contains following information:
•
Recipient: Email address of the recipient
•
Connections: Number of connections to the recipient
 
To view the recipient details for the selected sender, sender host and mail virus, drill down by clicking the recipient in the graph or hyperlink in the table.
Top Receiver Hosts widget
Widget report displays number of connections per host (receiver) by the selected mail virus and sender host.
 
View report from Reports → Virus → Mail Viruses → Virus → Top Sender Host widget → Sender Host.
 
Bar graph displays number of connections per host(receiver) while tabular report contains following information:
•
Receiver Host: IP address of the receiver host
•
Connections: Number of connections to the receiver host
 
To view the receiver host details for the selected receiver host, sender host and mail virus, drill down by clicking the receiver host in the graph or hyperlink in the table.
Top Applications widget
Widget report displays number of connections per application by the selected mail virus and sender host.
 
View report from Reports → Virus → Mail Viruses → Virus → Top Sender Hosts widget → Sender Host.
 
Bar graph displays number of connections per application while tabular report contains following information:
•
Application: Displays name of the application as defined in Cyberoam iView. If application is not defined in Cyberoam iView then this field will display application identifier as combination of protocol and port number. To define an undefined application or group an unassigned application please refer to Add Custom Application
•
Connections: Number of connections to the application
 
To view the application details for the selected application, sender host and mail virus, drill down by clicking the application in the graph or hyperlink in the table.
Top Users widget
Widget report displays number of connections per user by the selected mail virus and sender host.
 
View report from Reports → Virus → Mail Viruses → Virus → Top Sender Hosts widget → Sender Host.
 
Bar graph displays number of connections per user while tabular report contains following information:
•
User: Username of the user as defined in the monitored device. If username is not defined in the monitored device then it will be considered as traffic generated by ‘Unknown’ user
•
Connections: Number of connections to the application
 
To view the user details for the selected user, sender host and mail virus, drill down by clicking the username in the graph or hyperlink in the table.
